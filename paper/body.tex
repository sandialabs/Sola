\section{Introduction}

TODO.

\begin{purpose}
\begin{itemize}
\item \textbf{Motivation}: Modern computational problems go beyond forward simulation and require outer-loop analyses (inverse problems, optimal experimental design, sensitivity analysis, optimization, uncertainty quantification, etc.)
\item \textbf{Challenge}: Outer-loop problems are mathematically complicated (many moving pieces) and computationally demanding.
\item \textbf{Challenge}: There do exist algorithms and approaches for solving outer-loop problems computationally, but such algorithms are difficult to implement at scale within HPC frameworks. HPC has high software complexity, lots of minutia.
\item \textbf{Solution}: This paper presents two complementary software libraries that aim to lower the barrier to entry for practitioners who need to solve outer-loop problems. The first is a MATLAB library that implements the algorithms in a mathematically intuitive way and provides templates for interfacing with the algorithmic tools. The second is a \cpp library with a similar structure as the \MATLAB library but with state-of-the-art HPC features (parallelism, etc.).
\item Two representative demonstrations: PDE-constrained optimization and HDSA.
\item Final example cranks up the computational difficulty to 11.
\item Outline the paper.
\end{itemize}
\end{purpose}
